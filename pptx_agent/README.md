# Manus-Style PPTX Generation Agent

This is an independent PPTX agent that follows the workflow you described:

1. Take a simple prompt such as `Create a 15-slide pitch deck for our AI platform`.
2. Research the topic first when a search provider is configured.
3. Create `pitch_deck_structure.txt`.
4. Create detailed `slide_content.md`.
5. Render an HTML, page-by-page slide preview.
6. Convert the same slide plan into a downloadable `.pptx` when the user clicks download.

The app intentionally uses only the Python standard library, so it works in the current workspace without installing packages.

## Run

Frontend bundle is required (built into `pptx_agent/web/dist/`).

First time / after frontend changes:

```bash
cd /home/mamun/projects/pptx-generation-agent/pptx_agent/frontend
npm install         # one-time, requires Node 20+
npm run build       # outputs to ../web/dist/
```

Then start the backend (serves the built bundle):

```bash
cd /home/mamun/projects/pptx-generation-agent/pptx_agent
python3 run.py
```

Open:

```text
http://127.0.0.1:8787
```

### Frontend HMR (dev)

```bash
cd /home/mamun/projects/pptx-generation-agent/pptx_agent/frontend
npm run dev         # Vite on :5173, proxies /api/* to :8787
```

Keep `python3 run.py` running in another terminal so API calls succeed.

## Optional `.env`

```bash
cp .env.example .env
```

Add an LLM key if you want the slide strategy and copy generated by a model.

The local `.env` can be used without an LLM key:

```bash
LLM_API_KEY=
LLM_MODEL=
SEARCH_PROVIDER=searxng
SEARXNG_URL=http://127.0.0.1:8890
```

## Local SearXNG

Start the project-local SearXNG container:

```bash
cd /home/mamun/projects/pptx-generation-agent
docker compose -f docker-compose.searxng.yml up -d
```

Check the JSON endpoint:

```bash
curl "http://127.0.0.1:8890/search?q=healthcare%20bangladesh&format=json"
```

Stop it when needed:

```bash
docker compose -f docker-compose.searxng.yml down
```

For search, the agent now prefers local SearXNG when `SEARCH_PROVIDER=auto`.
Deep mode searches multiple angles from the user's prompt, keeps a few results
per angle, and fetches short text excerpts from the top source pages when they
are reachable:

```bash
SEARCH_PROVIDER=auto
SEARCH_DEPTH=deep
SEARXNG_URL=http://127.0.0.1:8890,http://127.0.0.1:8888,http://127.0.0.1:8080
MAX_SEARCH_RESULTS=18
MAX_SEARCH_QUERIES=7
MAX_RESULTS_PER_QUERY=4
```

If your SearXNG runs on another port, change `SEARXNG_URL`. It can be a single URL or a comma-separated fallback list. You can also force it with `SEARCH_PROVIDER=searxng`. Without a working search provider, the agent still runs with deterministic local research fallback and records that in the activity log.

## Output

Generated runs are written to:

```text
pptx_agent/output/<job-id>/
```

Each run includes:

- `pitch_deck_structure.txt`
- `slide_content.md`
- `slides.html`
- `deck.json`
- `deck.pptx` after the download endpoint is used
